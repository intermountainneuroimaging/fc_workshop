Big picture: what this tutorial is trying to accomplish

The goal is to take raw resting-state fMRI data and produce a cleaned 4D BOLD file that you can use for functional connectivity analyses. Functional connectivity asks whether the BOLD signal in one brain region rises and falls with the BOLD signal in another brain region across time. To do that responsibly, the data need to be organized, readable, motion-corrected, spatially aligned, and cleaned enough for time-series analysis.

Where we start

We begin with OpenNeuro Midnight Scan Club data in a BIDS-like folder structure. The important files for this tutorial are anatomical T1w images, resting-state BOLD files, and one glasslexical task BOLD run that can be used later for PPI examples.

The T1w image gives us a high-resolution structural reference. The resting BOLD file gives us a 4D time series for functional connectivity. The task BOLD file gives us a later example where connectivity can be modeled in relation to task events.

Where we end

The main endpoint is bold_fc_ready_mni.nii.gz. This is a cleaned 4D BOLD file in MNI space.

Once that file exists, you can extract seed time series, average signal within atlas parcels, compute correlation matrices, and move into graph theory analyses.

Why HCP-style folders?

The HCP pipelines expect a particular folder style, especially folders such as unprocessed/3T, T1w, and MNINonLinear. Even when this tutorial uses a practical FSL route for the MSC subset, keeping an HCP-style structure makes the workflow easier to understand and easier to extend later.

Why the MSC path is practical rather than fully official

The sample MSC download used here gives us T1w, resting-state BOLD files, and one task BOLD run, but it does not provide the full set of T2w, SBRef, and distortion-correction inputs that the official full HCP minimal preprocessing path expects. The tutorial therefore uses a practical HCP-style/FSL workflow for the MSC sample data and keeps a fuller HCP template for datasets that do have the required inputs.

1. Install AWS CLI in scratch

This installs AWS CLI v2 and avoids the interactive unzip overwrite problem.

What this step is doing

The AWS CLI is a command-line tool that can copy files from public S3 buckets. OpenNeuro exposes datasets through public S3, so this lets us download only the files we need instead of clicking through a browser.

Why we install it in scratch

You usually do not have permission to install software globally on the server. Installing into /scratch/alpine/<user>/software gives each person a local copy they can run.

Important: the script removes old aws installer files before unzipping. This prevents the interactive prompt that asks whether to replace files. 
Expected useful output:
AWS CLI installed:
/scratch/alpine/jade6100/software/bin/aws
aws-cli/2.x.x ...
Create and run 01_install_aws_cli.sh
mkdir -p /scratch/alpine/__USER__/openneuro/hcp_scripts
cat > /scratch/alpine/__USER__/openneuro/hcp_scripts/01_install_aws_cli.sh <<'BASH'
#!/usr/bin/env bash
set -euo pipefail

###############################################################################
# 01_install_aws_cli.sh
# Install AWS CLI v2 into scratch.
#
# Why this exists:
#   The OpenNeuro Midnight Scan Club data are downloaded from public S3.
#   You do not need an AWS account, but you do need the AWS CLI command.
###############################################################################

export USERNAME="${USERNAME:-jade6100}"
export SOFTWARE_DIR="${SOFTWARE_DIR:-/scratch/alpine/${USERNAME}/software}"
export BIN_DIR="${BIN_DIR:-${SOFTWARE_DIR}/bin}"
export AWS_INSTALL_DIR="${AWS_INSTALL_DIR:-${SOFTWARE_DIR}/aws-cli}"

mkdir -p "${SOFTWARE_DIR}" "${BIN_DIR}"
cd "${SOFTWARE_DIR}"

echo "Installing AWS CLI into:"
echo "  ${AWS_INSTALL_DIR}"
echo "Binary link will be:"
echo "  ${BIN_DIR}/aws"
echo

# Remove old installer files so unzip does not ask:
# replace aws/README.md? [y]es, [n]o, [A]ll...
rm -rf aws awscliv2.zip

echo "Downloading AWS CLI v2 installer..."
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o awscliv2.zip

echo "Unzipping AWS CLI installer..."
unzip -q awscliv2.zip

echo "Running AWS installer..."
./aws/install \
  --install-dir "${AWS_INSTALL_DIR}" \
  --bin-dir "${BIN_DIR}" \
  --update

export PATH="${BIN_DIR}:${PATH}"

echo
echo "AWS CLI installed:"
which aws
aws --version
BASH
chmod +x /scratch/alpine/__USER__/openneuro/hcp_scripts/01_install_aws_cli.sh

export USERNAME="__USER__"
bash /scratch/alpine/__USER__/openneuro/hcp_scripts/01_install_aws_cli.sh

2. Download Midnight Scan Club subjects

This downloads five subjects by default. It pulls resting-state BOLD for functional connectivity and one glasslexical task run for later PPI examples. You can change the subject list above.

Why we use include and exclude rules

The full dataset is larger than what you need for this tutorial. The command first excludes everything, then includes only dataset metadata, T1w anatomy, rest BOLD files, one glasslexical task BOLD run, JSON sidecars, event files, and scan logs.

What the JSON sidecars do

JSON files store acquisition metadata. For BOLD data, this often includes timing or sequence information. Even when a script does not use every JSON field, downloading sidecars keeps the dataset interpretable.

Why public S3 works: the command uses --no-sign-request, which tells AWS CLI not to look for private credentials.

3. Verify the MSC download

This block fixes the path issue you hit. It uses /scratch/..., not $/scratch/....

Why verification comes before preprocessing

Preprocessing errors are hard to interpret if we do not first know whether the files exist, whether the path is correct, and whether FSL can read the images. Verification separates download problems from processing problems.

What fslinfo tells us

fslinfo prints image dimensions, voxel size, number of volumes, data type, and TR-related header information. For a BOLD file, dim4 is the number of time points.

What good output looks like: the current folder should be /scratch/alpine/<user>/openneuro/ds000224, not a folder under /home/<user>/$/scratch.
Create and run 03_verify_msc_download.sh
mkdir -p /scratch/alpine/__USER__/openneuro/hcp_scripts

4. Load the HCP/FSL environment

This uses the HCP Pipeline, FSL, FreeSurfer, and Workbench paths.

Why environment setup matters

Most neuroimaging tools are command-line programs. The shell needs to know where those programs live. Updating PATH makes commands like fslinfo, mcflirt, and flirt visible.

Why FreeSurfer needs special handling

FreeSurfer setup scripts can contain internal checks that are harmless interactively but can stop strict shell scripts. The temporary set +e, set +u, and set +o pipefail block lets FreeSurfer load cleanly, then turns strict behavior back on.

HCP tools used here: this tutorial checks for HCP pipeline scripts and Workbench, but the MSC practical route uses FSL commands for the sample dataset because the full set of HCP inputs is not present in the downloaded subset.

5. Process MSC data to FC-ready files

This produces cleaned 4D resting-state BOLD files in MNI space. The guide also downloads one raw glasslexical task run for the later PPI exercise, but Section 5 only processes the resting-state run into an FC-ready file.

Processing choices used in this script: the script drops the first 5 volumes by default, uses TR=2.2 seconds, applies FSL -bptf 22.7 2.27, and uses linear FLIRT registration. With this TR, the temporal filter is approximately 0.01 to 0.10 Hz, because 22.7 volumes corresponds to about a 100-second high-pass cutoff and 2.27 volumes corresponds to about a 10-second low-pass cutoff.
Processing choiceCurrent valueWhere it is setWhat it means

Preflight: the script checks the raw BIDS folder, creates output folders, confirms core FSL commands exist, and counts T1w and BOLD files.
Stage inputs: the T1w and BOLD files are copied into an HCP-style folder layout. This keeps raw downloaded data separate from staged processing inputs.
Structural prep: the T1w image is cropped, brain-extracted, and registered to MNI space. This gives the functional data a structural bridge to standard space.
Functional prep: the first 5 volumes are dropped by default using fslroi, a mean image is created, a brain mask is estimated, and motion correction is run.
Registration: the script uses linear registration. The T1w-to-MNI step uses 12 DOF affine FLIRT, the BOLD-to-T1w step uses 6 DOF rigid-body FLIRT, and the combined transform moves the BOLD time series into MNI space.
Cleanup and filtering: motion parameters are regressed out when available, fslmaths -bptf 22.7 2.27 applies an approximate 0.01 to 0.10 Hz temporal filter for TR=2.2 seconds, the MNI brain mask is enforced, and the final FC-ready file is written.
Interpretation note: this is a practical workflow for volume-based FC. It is not claiming to replace a full official HCP minimal preprocessing run when a dataset includes T2w, SBRef, and distortion-correction inputs.

#!/usr/bin/env bash
set -euo pipefail

###############################################################################
# 05_msc_hcp_style_to_fc_ready.sh
# Prepare MSC resting-state data for volume-based functional connectivity.
#
# Output:
#   ${FC_DIR}/${SUBJECT}/${SESSION}/bold_fc_ready_mni.nii.gz
###############################################################################

export USERNAME="${USERNAME:-jade6100}"
export RAW_BIDS="${RAW_BIDS:-/scratch/alpine/${USERNAME}/openneuro/ds000224}"
export STUDY_DIR="${STUDY_DIR:-/scratch/alpine/${USERNAME}/openneuro/hcp_outputs}"
export FC_DIR="${FC_DIR:-/scratch/alpine/${USERNAME}/openneuro/fc_ready}"

export SUBJECTS="${SUBJECTS:-MSC01}"
export FUNC_SESSIONS="${FUNC_SESSIONS:-ses-func01}"

export HIGHPASS_SIGMA="${HIGHPASS_SIGMA:-22.7}"
export LOWPASS_SIGMA="${LOWPASS_SIGMA:-2.27}"

# Processing choices:
#   DROP_TRS=5 removes the first five BOLD volumes.
#   TR=2.2 seconds is used to interpret the temporal filter.
#   FSL -bptf expects sigma values in volumes, not Hz.
#   HIGHPASS_SIGMA=22.7 gives an approximate 100-second high-pass cutoff, about 0.01 Hz.
#   LOWPASS_SIGMA=2.27 gives an approximate 10-second low-pass cutoff, about 0.10 Hz.
#   Registration is linear only in this practical MSC route.

export FSLDIR="${FSLDIR:-/projects/ics/software/fsl/6.0.7}"
export FREESURFER_HOME="${FREESURFER_HOME:-/projects/ics/software/freesurfer/6.0.1}"
export HCPPIPEDIR="${HCPPIPEDIR:-/projects/ics/software/hcp_pipeline/HCP_Pipelines-3.4.0}"

source "${FSLDIR}/etc/fslconf/fsl.sh"

set +o pipefail

  echo "ERROR: FreeSurfer setup failed."
  exit 1
fi

MNI_BRAIN="${FSLDIR}/data/standard/MNI152_T1_2mm_brain.nii.gz"
MNI_MASK="${FSLDIR}/data/standard/MNI152_T1_2mm_brain_mask.nii.gz"

process_subject_session() {
  local sub="$1"
  local ses="$2"
  local bids_sub="sub-${sub}"
  local run_label
  run_label=$(echo "${ses}" | sed 's/ses-func//')
  local fmri_name="rfMRI_REST${run_label}_LR"

  local hcp_sub_dir="${STUDY_DIR}/${sub}"
  local unproc_dir="${hcp_sub_dir}/unprocessed/3T"
  local t1w_unproc="${unproc_dir}/T1w_MPR1/${sub}_3T_T1w_MPR1.nii.gz"
  local fmri_unproc_dir="${unproc_dir}/${fmri_name}"
  local bold_unproc="${fmri_unproc_dir}/${sub}_3T_${fmri_name}.nii.gz"

  local t1w_src
  t1w_src=$(find "${RAW_BIDS}/${bids_sub}" -path "*/anat/*_T1w.nii.gz" | sort | head -1 || true)
  local bold_src="${RAW_BIDS}/${bids_sub}/${ses}/func/${bids_sub}_${ses}_task-rest_bold.nii.gz"

  if [[ -z "${t1w_src}" ]]; then
    echo "ERROR: no T1w found for ${bids_sub}"
    return 1
  fi

  if [[ ! -f "${bold_src}" ]]; then
    echo "WARNING: missing BOLD, skipping ${bids_sub} ${ses}"
    return 0
  fi

  echo
  echo "============================================================"
  echo "PROCESSING ${sub} ${ses}"
  echo "============================================================"

  mkdir -p "${unproc_dir}/T1w_MPR1" "${fmri_unproc_dir}"

  echo "Staging T1w:"
  echo "${t1w_src}"
  cp -f "${t1w_src}" "${t1w_unproc}"

  echo "Staging BOLD:"
  echo "${bold_src}"
  cp -f "${bold_src}" "${bold_unproc}"

  local t1w_dir="${hcp_sub_dir}/T1w"
  mkdir -p "${t1w_dir}"

  echo "Structural prep..."
  cp -f "${t1w_unproc}" "${t1w_dir}/T1w.nii.gz"
  robustfov -i "${t1w_dir}/T1w.nii.gz" -r "${t1w_dir}/T1w_acpc_dc.nii.gz"
  bet "${t1w_dir}/T1w_acpc_dc.nii.gz" "${t1w_dir}/T1w_acpc_dc_brain.nii.gz" -R -f 0.25 -g 0 -m
  flirt -in "${t1w_dir}/T1w_acpc_dc_brain.nii.gz" -ref "${MNI_BRAIN}" -omat "${t1w_dir}/T1w_to_MNI.mat" -out "${t1w_dir}/T1w_acpc_dc_brain_mni.nii.gz" -dof 12
  convert_xfm -omat "${t1w_dir}/MNI_to_T1w.mat" -inverse "${t1w_dir}/T1w_to_MNI.mat"

  local out_dir="${FC_DIR}/${sub}/${ses}"
  mkdir -p "${out_dir}"

  echo "Functional prep..."
  echo "Dropping the first ${DROP_TRS} volumes before motion correction."
  fslroi "${bold_unproc}" "${out_dir}/bold_drop.nii.gz" "${DROP_TRS}" -1
  fslmaths "${out_dir}/bold_drop.nii.gz" -Tmean "${out_dir}/bold_mean.nii.gz"
  bet "${out_dir}/bold_mean.nii.gz" "${out_dir}/bold_mean_brain.nii.gz" -f 0.3 -m
  mcflirt -in "${out_dir}/bold_drop.nii.gz" -out "${out_dir}/bold_mc.nii.gz" -plots -mats -reffile "${out_dir}/bold_mean_brain.nii.gz"
  fslmaths "${out_dir}/bold_mc.nii.gz" -mas "${out_dir}/bold_mean_brain_mask.nii.gz" "${out_dir}/bold_mc_brain.nii.gz"

  echo "Registering BOLD to T1w and MNI using linear FLIRT transforms..."
  echo "BOLD-to-T1w: 6 DOF rigid-body linear registration"
  echo "T1w-to-MNI: 12 DOF affine linear registration"
  flirt -in "${out_dir}/bold_mean_brain.nii.gz" -ref "${t1w_dir}/T1w_acpc_dc_brain.nii.gz" -omat "${out_dir}/bold_to_T1w.mat" -out "${out_dir}/bold_mean_to_T1w.nii.gz" -dof 6
  convert_xfm -omat "${out_dir}/bold_to_MNI.mat" -concat "${t1w_dir}/T1w_to_MNI.mat" "${out_dir}/bold_to_T1w.mat"
  flirt -in "${out_dir}/bold_mc_brain.nii.gz" -ref "${MNI_HEAD}" -applyxfm -init "${out_dir}/bold_to_MNI.mat" -out "${out_dir}/bold_mc_mni.nii.gz" -interp trilinear

  echo "Motion cleanup..."
  local par="${out_dir}/bold_mc.nii.gz.par"
  if [[ ! -f "${par}" ]]; then
    par="${out_dir}/bold_mc.par"
  fi

  if [[ -f "${par}" ]]; then
    cp -f "${par}" "${out_dir}/motion.par"
    fsl_regfilt -i "${out_dir}/bold_mc_mni.nii.gz" -d "${out_dir}/motion.par" -f "1,2,3,4,5,6" -o "${out_dir}/bold_mc_mni_motionreg.nii.gz"
  else
    echo "WARNING: motion parameter file not found. Continuing without motion regression."
    cp -f "${out_dir}/bold_mc_mni.nii.gz" "${out_dir}/bold_mc_mni_motionreg.nii.gz"
  fi

  echo "Temporal filtering..."
  echo "Applying FSL -bptf ${HIGHPASS_SIGMA} ${LOWPASS_SIGMA}. With TR=${TR}, this is approximately 0.01 to 0.10 Hz."
  fslmaths "${out_dir}/bold_mc_mni_motionreg.nii.gz" \
    -bptf "${HIGHPASS_SIGMA}" "${LOWPASS_SIGMA}" \
    "${out_dir}/bold_fc_ready_mni.nii.gz"

  fslmaths "${out_dir}/bold_fc_ready_mni.nii.gz" -mas "${MNI_MASK}" "${out_dir}/bold_fc_ready_mni.nii.gz"
  fslmaths "${out_dir}/bold_fc_ready_mni.nii.gz" -Tmean "${out_dir}/bold_fc_ready_mni_mean.nii.gz"
  fslnvols "${out_dir}/bold_fc_ready_mni.nii.gz" > "${out_dir}/n_volumes_after_drop.txt"

  cat > "${out_dir}/processing_parameters.txt" <<PARAMS
DROP_TRS=${DROP_TRS}
TR=${TR}
HIGHPASS_SIGMA=${HIGHPASS_SIGMA}
LOWPASS_SIGMA=${LOWPASS_SIGMA}
APPROX_FILTER_HZ=0.01_to_0.10_when_TR_is_2.2_seconds
REGISTRATION=T1w_to_MNI_linear_FLIRT_12DOF;BOLD_to_T1w_linear_FLIRT_6DOF;combined_linear_transform_to_MNI
NONLINEAR_REGISTRATION=not_used_in_this_practical_MSC_route
PARAMS

  echo "Finished ${sub} ${ses}"
  echo "Output: ${out_dir}/bold_fc_ready_mni.nii.gz"
}

Troubleshooting guide

ProblemLikely causeWhat to do
aws: command not foundAWS CLI is not installed or the scratch bin folder is not in PATH.Run the AWS install section again, then run which aws.
BrokenPipeError after aws s3 ls | headhead closed the output stream early.The tutorial disables pipefail only for that check. It is not a data problem.
Verifier finds zero filesThe download did not run, the subject list is wrong, or the path points to the wrong folder.Run pwd, then find /scratch/alpine/<user>/openneuro/ds000224 -name "*task-rest_bold.nii.gz" | head.
fslinfo: command not foundFSL is not loaded or FSLDIR is wrong.Run the HCP/FSL environment section and check which fslinfo.
Folder starts with /home/<user>/$/scratchA dollar sign was accidentally pasted before an absolute path.Use the cleanup section and rerun commands with paths that begin directly with /scratch.
Registration output looks wrongBrain extraction or alignment may have failed, or the anatomical image has unusual coverage.Inspect mean images and registration outputs visually before using FC results.
